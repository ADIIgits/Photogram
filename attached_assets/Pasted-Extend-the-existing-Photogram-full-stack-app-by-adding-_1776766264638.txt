Extend the existing **Photogram** full-stack app by adding a **location-based smart search suggestion system**.

---

## 🎯 Feature Overview

Implement a search bar that:

* Shows suggestions when focused
* Suggestions are **location-aware**
* Ranked by:

  * proximity to user
  * popularity (user clicks)

---

## 🧱 Tech Additions

* PostgreSQL (with optional PostGIS support)
* Redis (for caching suggestions)
* Backend: Node.js + Express
* Frontend: React + Tailwind

---

## 🗄️ Database Design

### Table: Suggestion

Fields:

* id (UUID)
* text (string, indexed)
* lat (float)
* lng (float)
* click_count (integer, default 0)
* created_at (timestamp)

Indexes:

* index on (lat, lng)
* index on text

---

## ⚙️ Backend Features

### 1. Get Suggestions (on search bar focus)

Endpoint:
GET /search/suggestions?lat=..&lng=..

Logic:

* Check Redis cache first (key: suggestions:{lat}:{lng})
* If not cached:

  * Fetch suggestions within radius (e.g., 50km)
  * Sort by:

    * nearest distance
    * highest click_count
  * Cache result in Redis (TTL: 5–10 minutes)

Return:

* top 10 suggestions

---

### 2. Click Suggestion

Endpoint:
POST /search/suggestions/:id/click

Logic:

* Increment click_count in DB
* Optionally update Redis cache

---

### 3. Save New Search

Endpoint:
POST /search

Body:

* text
* lat
* lng

Logic:

* Ignore if text length < 3
* Check if similar suggestion exists nearby:

  * if yes → do nothing OR increment weight
  * if no → insert new suggestion

---

## 📐 Distance Calculation

If PostGIS available:

* Use ST_Distance

Else:

* Use Haversine formula
