Build a full-stack social media web app called **Photogram** for photographers.

## 🧱 Tech Stack

* Frontend: React (Vite) + TailwindCSS
* Backend: Node.js + Express
* Database: PostgreSQL (via Prisma ORM)
* Auth: JWT (access + refresh tokens)
* File Uploads: Cloudinary (or mock if keys not provided)
* State: React Query (TanStack Query)

---

## 🎯 Core Features

### 1. Authentication

* Signup / Login / Logout
* JWT-based auth
* Protected routes
* Endpoint:

  * POST /auth/signup
  * POST /auth/login
  * POST /auth/refresh
  * GET /auth/me

---

### 2. Users

* View profile
* Update profile
* Follow / Unfollow users

Fields:

* id
* name
* email
* avatar_url
* created_at

Endpoints:

* GET /users/:id
* PATCH /users/:id
* POST /users/:id/follow
* DELETE /users/:id/follow

---

### 3. Posts

Each post contains:

* image
* title
* caption
* optional camera reference
* created_at

Endpoints:

* GET /posts (global feed)
* GET /posts/:id
* POST /posts
* PATCH /posts/:id
* DELETE /posts/:id

---

### 4. Likes

* Like / Unlike post
* Prevent duplicate likes

Endpoints:

* POST /posts/:id/like
* DELETE /posts/:id/like
* GET /posts/:id/likes

---

### 5. Comments

* Add comment
* Delete comment
* List comments

Fields:

* id
* user_id
* post_id
* content
* created_at

Endpoints:

* GET /posts/:id/comments
* POST /posts/:id/comments
* DELETE /comments/:id

---

### 6. Camera System

Predefined camera list (admin-controlled)

Fields:

* id
* name
* icon_url

Endpoints:

* GET /cameras
* GET /cameras/:id
* POST /cameras (admin only)
* PATCH /cameras/:id (admin only)
* DELETE /cameras/:id (admin only)

---

### 7. Feed System

* Personalized feed:

  * Posts from followed users
  * Sorted by latest
* Explore (/discover):

  * Random + trending posts (based on likes)

Endpoints:

* GET /feed
* GET /discover

---
